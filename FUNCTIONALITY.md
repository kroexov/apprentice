# Golang Apprentice — функционал

Чек-лист всего, что должен поддерживать бэкенд. Без эндпоинтов — только бизнес-логика и доменная модель.

---

## 1. Сущности

### Stage (этап)
- `id` (slug)
- `order` — порядковый номер
- `title` — полное название
- `short_title` — краткое (для карточек)
- `description` — описание
- `max_score` — максимум баллов за этап (по умолчанию 10)

Стартовый набор:
1. Первый проект
2. Программирование в Word
3. Первый PR
4. Первый читаемый PR
5. Свой пет-проект

### Candidate (кандидат)
- `id`
- `name` — ФИО
- `handle` — техлогин (`ivan.sokolov`), уникальный
- `city`
- `age`
- `bio` — текст био
- `avatar_color` — цвет фона аватарки
- `initials` — 2 символа для аватарки
- `avatar_url` — опционально, картинка
- `strengths[]` — теги «сильные стороны»
- `weaknesses[]` — теги «зоны роста»
- `current_stage_id` — где сейчас
- `created_at`, `updated_at`

### StageScore (оценка за пройденный этап)
- `id`
- `candidate_id`
- `stage_id`
- `score` (1..10)
- `scored_at`
- `scored_by` (опционально, если будет авторизация)

Уникальность: `(candidate_id, stage_id)` — один кандидат, одна оценка за этап.

### Cohort / Сезон (опционально)
В шапке указано «Сезон 1 · Когорта весна-лето 2026». Если будет несколько потоков — отдельная сущность `Cohort` и FK у `Candidate`.

## 2. Бизнес-правила

### Прогресс
- Кандидат всегда стоит на одном текущем этапе.
- За все этапы с `order < current.order` существуют `StageScore`.
- На текущем этапе `StageScore` ещё нет.
- Кандидат завершил программу, когда у него есть `StageScore` по всем этапам.

### Перевод на следующий этап (главное действие)
1. Создать `StageScore` для текущего этапа с указанным баллом (1..10).
2. Передвинуть `current_stage_id` на следующий этап по `order`.
3. Если текущий — последний: либо оставить указатель на последнем и считать программу завершённой по числу оценок, либо проставить `completed_at`.

### Подсчёт суммарного балла
- `total_points = sum(StageScore.score)` по кандидату.
- `max_points = count(stages) * max_score`.

### Корректировка оценки (опционально)
Поменять уже выставленный `StageScore.score` (админская правка) — без перевода этапа.

### Откат на предыдущий этап (опционально)
Удалить последний `StageScore`, передвинуть `current_stage_id` на шаг назад.

## 3. Чтение

### Список кандидатов
Каждый — с агрегатами: `current_stage`, `total_points`, `max_points`, `completed_stages`, `stage_count`.

Сортировка в трёх режимах:
- по сумме баллов (по убыванию)
- по текущему этапу (по убыванию `order`)
- по имени (А–Я, локаль `ru`)

### Детальная карточка кандидата
- Все базовые поля (имя, био, теги, город, возраст).
- Текущий этап.
- Сумма / максимум баллов.
- История по всем этапам: для каждого — статус (`done` / `current` / `todo`), оценка (если есть), дата проставления.

### Группировка по канбану
Кандидаты разложены по `current_stage_id` — массив на каждый этап.

### Сводка для шапки
- общее число кандидатов
- общее число этапов
- сумма всех баллов всех кандидатов / максимум
- средний этап (среднее `current.order`, дробное)

---

## 4. CRUD

### Кандидаты
- создать, обновить базовые поля, удалить.

### Этапы
- создать, обновить, удалить (с проверкой: нельзя удалить этап, у которого есть оценки).
- переупорядочить (сменить `order`).

---

## 5. Валидации

- `score`: целое, 1..`stage.max_score`.
- `name`: непусто, ≤ 80.
- `handle`: уникальный, `^[a-z0-9.\-_]{2,40}$`.
- `initials`: 1–3 символа.
- `age`: 14..120.
- `strengths`/`weaknesses`: до 10 шт, каждый ≤ 40 символов.
- Запрет повторного `advance` по одному и тому же `(candidate, stage)`.
- Нельзя сделать `advance`, если кандидат уже завершил все этапы.

---

## 6. Аутентификация (на вырост)

- Роли: `viewer` (только чтение), `mentor` (advance + rate), `admin` (CRUD кандидатов и этапов).
- В `StageScore.scored_by` пишем, кто поставил оценку.

---

## 7. Сидинг для тестов

5 кандидатов, 5 этапов. Каждому — случайный `current_stage` (0..4), и для всех уже пройденных этапов случайный `score` (5..10). Сейчас фронт делает это через `Math.random()` при загрузке — на бэке нужно положить в фикстуры/миграцию, без рандома на каждом запросе.
